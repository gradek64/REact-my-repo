        Hi Ire,
        HTML parsing in the diagrams refers to tokenization of the provided HTML or DOM building?

        During the tokenization phase the browser's preloader will kick-in and initiate JS download even before DOM building starts, so the "script fetch" phase may potentially be shorter or even have 0 length.

        One more thing to note is that deffered scripts will always execute in the source order, no matter which downloads first. Async will execute immediately after its download.

                    There's a small grammatical error in the article:

                    "HTML documents are parsed in order, from the first opening <html> element to it's close."

                    The possessive form of the word "its" shouldn't have any apostrophe. Possessive pronouns like his, hers, yours, ours, theirs, whose, etc. never get the apostrophe. With the apostrophe, it's always a contraction of either "it is" or "it has". Hope this helps!

                                                        Ifeora Okechukwu • 2 years ago • edited

                                                        Great Article Ire!! I actually found out a couple more things in this line concerning "async" and "defer". Which is always use async and defer whenever you have scripts that do not have any direct bearing on page load functionality (e.g Google Analytics, BugSnag and Facebook Pixel). Specifically, don't use async / defer for scripts that depend on each other.

                                                        However, if you must then make sure you use defer only on IE9 and below with conditional comments and asyc only in other browsers and watch out for when they load/execute using "onreadystatechange" and "onload" for IE9- and other browsers respectively and signal to a custom JS event e.g. "onPageReady" when each and every script loads.

                                                        You can also bootup other parts of the app with "DOMContentLoaded" and/or "Load" events. Basically defer delays "DOMContentLoaded" event only in IE and Chrome but not in Firefox so hence not reliable cross-browser and the need for the custom event.

                                                        I got this from Paul Irish on a GitHub Issue thread (https://github.com/h5bp/laz.... Just thought to share. Once again nice Article!!!
